fix: Support UUID and non-numeric primary keys in feature pickers and spatial operations

CRITICAL FIX v4.0: Feature pickers, zoom, flash, and identify operations now work
correctly with all field types including UUID, VARCHAR, and TEXT primary keys.

Problem:
--------
When layers used non-numeric primary keys (UUID, VARCHAR, TEXT), the following
operations would fail:

1. Feature picker single/multiple selection
2. Zoom to selected features
3. Flash selected features  
4. Custom expressions with non-numeric PKs

The root cause was improper conversion of PK values to SQL expressions:
- UUID objects were not converted to strings before building SQL
- String values were not properly escaped (single quotes)
- Type detection relied only on field.isNumeric() without explicit string conversion

Symptoms:
- Invalid SQL expressions like: "id" = 'UUID('7b2e1a3e-...')' 
- Feature picker showing items but zoom/flash not working
- Expression validation errors in logs

Solution:
---------
Applied fixes in 3 critical zones:

1. **exploring_controller.py** (Line ~1603-1615)
   - Explicit str() conversion for all non-numeric PK values
   - Proper quote escaping: str(pk_value).replace("'", "''")
   - Applies to single selection AND list-based (IN clause) operations

2. **custom_widgets.py** (Line ~895-900)
   - Convert non-numeric fid values to strings when storing in list items
   - Ensures proper handling when building SQL expressions later
   - Prevents UUID object propagation through widget data

3. **expression_builder.py** (Line ~490-510)
   - Enhanced type detection using field.isNumeric()
   - Unified handling for UUID, text, and other non-numeric types
   - Consistent formatting: numeric unquoted, text/UUID quoted with escaping

Technical Details:
------------------
**Before Fix:**
```python
# ❌ Crashes with UUID objects
expression = f'"{pk_name}" = \'{pk_value}\''
# Result: "id" = 'UUID('7b2e1a3e-...')' ← INVALID
```

**After Fix:**
```python
# ✅ Works with all types
pk_value_str = str(pk_value).replace("'", "''")
expression = f'"{pk_name}" = \'{pk_value_str}\''  
# Result: "id" = '7b2e1a3e-...' ← VALID
```

**Expression Examples:**
- Numeric:  `"id" = 123`
- UUID:     `"id" = '7b2e1a3e-b812-4d51-bf33-7f0cd0271ef3'`
- VARCHAR:  `"code" = 'ABC123'`
- List:     `"id" IN ('uuid1', 'uuid2', 'uuid3')`
